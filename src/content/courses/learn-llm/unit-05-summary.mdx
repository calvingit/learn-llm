第五单元的核心线索是"从说对话到做成事"。Prompt 不是咒语而是上下文控制，工具调用让模型从只会说话变成能请求外部系统执行操作，Agent 把单次回答扩展成围绕目标的执行闭环，但 Agent 也会失控——目标漂移、重复循环、工具误用、伪造结果。

### 你应该已经掌握

- **Prompt 的本质是上下文控制**：背景、任务、边界、格式、检查——把这六件事说清楚，比华丽的措辞更有效
- **工具调用不是模型长出了手**：模型是调度台，外部系统才是执行者。权限校验、错误处理、审计日志必须由应用层实现，不能靠模型自觉
- **Agent 的核心是执行闭环**：思考→行动→观察→再决策→停止。没有停止条件的 Agent 只是昂贵的无限循环
- **能力和风险同步增长**：Agent 越能做事，越需要边界、权限、日志和人工确认。治理不是让 Agent 少做事，而是让它知道什么时候该停、该问人

### 关键直觉

把 Agent 想成一个能力很强但需要监督的实习生。它很适合做研究、整理、草稿这类"建议型闭环"，但在涉及发邮件、付款、删除数据等"执行型闭环"时，必须有人签字确认。
