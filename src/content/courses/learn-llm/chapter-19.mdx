## 核心问题

既然 Agent 能调用工具、分解任务、持续推进，为什么实际落地时经常跑偏？

因为每一步都是基于当前上下文做判断。只要目标、工具、结果或边界有一处不清楚，错误就可能被下一轮继续放大。

## 先建立直觉

一个实习生如果不知道什么时候该停止，就可能反复搜索资料，却迟迟不写结论。

Agent 也会遇到类似问题：它可能一直调用工具、忘记原始目标、误解错误信息，甚至编造一个并不存在的工具。

## 常见失控类型

- **工具滥用**：反复调用同一个工具，没有停止条件。
- **目标丢失**：做着做着忘记最初用户要什么。
- **工具幻觉**：编造不存在的工具或参数。
- **错误级联**：一次失败被误解，导致后续步骤越来越偏。

## 如何治理

可靠 Agent 不是只靠更强模型，还要靠系统边界：

- 限制工具权限。
- 设置最大步骤数。
- 明确停止条件。
- 对高风险动作要求用户确认。
- 记录每一步行动和结果，方便回溯。

## 一句话总结


> Agent 真正难的不是开始做事，而是安全、可控、可解释地停下来。
