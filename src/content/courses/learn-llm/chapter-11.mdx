## 本章核心问题

既然 Agent 能做事，为什么有时候会跑偏？

## 生活比喻

Agent 像一个很能干但容易自作主张的助手。

它能做很多事，但需要明确边界，需要检查结果，也需要知道什么时候停。

## 简单解释

Agent 每一步都在预测“下一步应该做什么”。如果中间一步判断错了，后面可能越走越偏。

常见问题包括：

- 理解错目标
- 调错工具
- 忽略限制
- 自己补充不存在的信息
- 一直循环，不知道停

## 真实使用场景

如果你让 Agent “帮我整理这个项目”，但没有说明哪些文件不能改、结果如何验证、什么时候停止，它可能会做出你没要求的修改。

目标越模糊，工具权限越大，越需要检查点。

## 小问题

一个稳定的 Agent 目标应该能回答：要做什么、不做什么、用什么工具、如何检查、什么时候算完成。

## 一句话总结

Agent 要稳定，不能只靠模型聪明，还需要清晰目标、明确步骤、工具限制和结果检查。
