## 核心问题

你用的 AI 助手不是"天生就聪明"的。它经历了一个漫长而系统的训练过程，从一个只会续写文字的"话痨"变成了可用的助手。

更值得问的是：为什么同样是"训练过的大模型"，有的写代码特别强，有的数学推理像数学家，有的则更擅长安全对话？答案在于训练的后半段——后训练（post-training）。

## 先建立直觉

训练一个大模型，像培养一个专业人才：

- 先准备大量学习材料（数据准备）
- 然后让他广泛阅读，建立知识基础（**预训练**）——此时他只是一个"什么都读过但不知道怎么用的毕业生"
- 接着进入**后训练**阶段：先教他按指令做事（监督微调），再让他通过反复练习和反馈变得专精（强化学习），同时规范什么该说什么不该说（偏好对齐）

后训练不是"附加项"。一个只经过预训练的模型只会续写文字，不会回答问题、不会写可运行的代码、也不会拒绝危险请求。它距离你用的 AI 助手还很远。

## 预训练：打地基

预训练的任务很简单：给模型海量文本，让它反复练习"预测下一个词"。这个过程就像一个人读了互联网上几乎所有公开的文字——书籍、文章、代码、论文、论坛讨论。

预训练结束后，你得到的是一个 **base model（基座模型）**。它能续写句子、能模仿各种文体，但它：

- **不会"回答问题"**——你问它"今天天气怎么样"，它可能续写成"今天天气怎么样呢？天气预报说明天……"
- **不会拒绝**——你让它写恶意代码，它也会顺着写下去
- **能力泛而不精**——它什么都见过，但什么都不特别擅长

预训练是四个阶段中消耗最大的。训练一个现代大模型可能需要数千张 GPU 连续运行数周到数月，电费可能高达数百万美元。

## 后训练：从"会说话"到"能干活"

后训练（post-training）是预训练之后所有训练步骤的总称。它的目标是把 base model 变成一个真正可用的助手。这个阶段包含三个关键步骤：

### 1. 监督微调（SFT）：学会"按指令做事"

给模型看大量"问题 → 理想答案"的范例。比如：

- 用户问："用 Python 读一个 CSV 文件"
- 理想回答：给出完整的代码示例，并解释每一步

经过成千上万个这样的高质量范例训练，模型学会了：当用户给出某种指令时，应该以什么样的格式和风格来回应。

这一阶段的本质是**教会模型对话的格式和基本行为模式**——不再续写，而是回答。

### 2. 强化学习（RL）：练出专精能力

这是当前大模型竞争中最重要的战场，也是大多数人容易忽略的一环。

监督微调只能让模型"模仿"好的回答。但如果想让模型在某个领域变得特别强——比如写出能正确运行的代码、解出高难度数学题、或者进行严密的逻辑推理——光靠模仿是不够的。

强化学习的方法是：**让模型反复尝试，做对了给奖励，做错了给惩罚。**

具体怎么做？以训练一个编程能力突出的模型为例：

- 给模型一道编程题
- 模型生成代码
- **自动运行代码，用测试用例检查结果**
- 通过的获得奖励信号，失败的获得惩罚信号
- 模型从这些信号中自己调整，逐步学会写出正确的代码

同样的方法可以用在数学上（验证答案是否正确）、推理上（验证逻辑链条是否自洽）。

这就是为什么不同公司的模型会有不同的"特长"：

| 模型            | 后训练重点                      |
| --------------- | ------------------------------- |
| DeepSeek-R1     | 推理 + 数学 RL，大规模强化学习  |
| Qwen-Coder 系列 | 代码生成 RL，配合自动化测试验证 |
| Claude          | 深度偏好对齐 + 推理训练         |
| GPT-4o          | 多模态 RL + 广泛领域对齐        |

一个模型是"写代码更强"还是"数学更强"，很大程度上取决于后训练阶段**在哪个方向上投入了更多的 RL 训练**，而不仅仅是预训练时看了多少相关数据。

### 3. 偏好对齐（RLHF）：知道什么该说、什么不该说

能力变强之后，还有一个关键问题：模型会顺着你的任何请求往下说吗？包括危险、违法或有害的内容？

偏好对齐的目标就是让模型学会"判断该不该说"和"怎么说更合适"。常用的方法是 **RLHF（从人类反馈中强化学习）**：

- 让人类标注员对模型的多个回答打分或排序
- 模型学习人类偏好——什么样的回答被认为是"有帮助、诚实、无害"的
- 通过强化学习将这种偏好内化为模型的行为准则

偏好对齐不是让模型变笨或什么都不敢说。它的目标是：**在保持能力的前提下，让模型在风险场景中成为更可靠的助手**。

## 为什么后训练比预训练更"划算"

预训练的成本极高——数千张 GPU 跑数周到数月，只有少数公司能承受。

后训练的成本远低于预训练，但收益巨大。一个已经预训练好的 base model，通过精心设计的后训练流程（高质量的 SFT 数据 + 针对性的 RL 训练），可以在特定能力上实现质的飞跃。

这就是为什么很多企业和研究机构（包括 DeepSeek、Qwen 团队）选择：**基于开源 base model 做后训练，而不是从头训练**。他们可以把资源和精力集中在让自己模型"专精"的后训练阶段。

## 一句话总结

> 大模型从"会说话"到"能干活"，关键在后训练。预训练打地基，SFT 教格式，RL 练专精，RLHF 定边界——四个步骤缺一不可，而 RL 阶段的能力训练是不同模型拉开差距的核心战场。
