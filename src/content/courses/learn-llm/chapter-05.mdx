## 核心问题

大模型看到的文字，和人类看到的一样吗？

不完全一样。人类会自然地看到词语、句子和段落。模型通常会先把文本拆成更小的单位，这些单位叫 Token。

## 先建立直觉

Token 可以理解为“文字积木”。模型不是一次吞下一整篇文章，而是把内容切成一块一块，再按顺序处理。

英文里，一个 Token 可能接近一个词或词的一部分。中文里，一个字、一个词的一部分、标点，都可能成为 Token。真实模型的拆分规则更复杂，不同模型也不完全一样。

## 为什么这很重要

Token 决定了三件事：

- 模型一次能读多少内容。
- 模型生成回答需要多少计算。
- API 或产品按量计费时大致如何计算。

当你把很长的文档、很多轮聊天记录、复杂格式要求都塞进 Prompt 时，模型看到的不是“一个需求”，而是一大串 Token。

## 常见误区

不要把 Token 简单等同于中文字数或英文单词数。它只是一个近似的处理单位。

在日常使用里，你不需要手动计算每个 Token。但你要知道：长文本不是免费进入模型的，长对话也不会无限保留。

## 一句话总结


> 大模型不是直接看整段话，而是用 Token 这套文字积木来读写内容。
